📖 采集模式说明

🔄

普通采集

日常使用
  • 触发方式：定时自动触发或手工点击"立即同步"
  • 停止条件：遇到已存在的重复数据时自动停止
  • 适用场景：日常增量同步，只获取新增的点赞
  • 404处理：未下载过的404回答会保存为删除状态记录
🚀

全量采集

首次推荐
  • 触发方式：仅手工点击"全量同步"按钮
  • 停止条件：穷尽用户的所有点赞记录为止
  • 适用场景：初次使用或需要完整备份时
  • 404处理：未下载过的404回答直接跳过（节省空间）

⚠️ 特殊情况说明

初次使用：

数据库为空时，普通采集等同于全量采集。建议直接使用"全量同步"以获得最佳效果。

404回答处理策略：
  • 本地已下载过：高亮标注删除状态，保留原内容，不会被404页面覆盖
  • 本地未下载过（全量）：直接跳过，节省存储空间
  • 本地未下载过（普通）：保存删除状态记录（用于数据完整性）

📥 浏览器扩展安装

  1. 下载扩展：点击下载
  2. 解压到本地文件夹
  3. 打开 Chrome/Edge，访问 chrome://extensions
  4. 开启"开发者模式"
  5. 点击"加载已解压的扩展程序"
  6. 选择解压后的文件夹完成安装

📖 如何获取用户 ID

  1. 打开知乎并登录 zhihu.com
  2. 点击头像 → 我的主页
  3. 查看地址栏 URL: zhihu.com/people/xxxxx
  4. xxxxx 就是用户 ID
💡 示例: zhang-san-123、wang-wu
